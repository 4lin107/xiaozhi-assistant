# AI语音助手Windows用户指南

## 一、关于获取APK文件

很抱歉，由于Buildozer（Kivy应用打包工具）的限制，**在Windows系统上直接构建Android APK文件是非常困难的**，通常需要Linux环境或WSL（Windows Subsystem for Linux）。

### 获取APK文件的替代方法

#### 方法1：使用Windows上的WSL（推荐）

虽然您提到没有WSL，但这是获取APK文件最可靠的方法。以下是安装WSL和Ubuntu的简单步骤：

1. **安装WSL**：
   - 打开PowerShell（管理员权限）
   - 运行命令：`wsl --install`
   - 安装完成后重启电脑

2. **设置Ubuntu**：
   - 重启后会自动打开Ubuntu安装界面
   - 创建用户名和密码

3. **在Ubuntu中构建APK**：
   - 进入项目目录：`cd /mnt/c/Users/[您的用户名]/Documents/trae_projects/AI`
   - 安装依赖：`pip3 install buildozer`
   - 运行打包命令：`buildozer -v android debug`
   - 构建完成后，APK文件将位于`dist/`目录下

#### 方法2：使用在线打包服务

如果您不想安装WSL，可以尝试使用第三方在线打包服务：

- **Google Colab**：可以在浏览器中运行Linux环境
- **App Inventor**：简单的Android应用创建平台

但这些方法需要更多的配置和学习成本。

## 二、直接在Windows上运行语音助手

您可以直接在Windows电脑上运行语音助手，不需要APK文件：

### 方法1：使用批处理脚本（推荐）

我们已经为您创建了一个`run_on_windows.bat`批处理文件，只需双击运行即可：

1. 双击`run_on_windows.bat`文件
2. 脚本会自动：
   - 检查Python是否已安装
   - 安装所需依赖
   - 启动语音助手

### 方法2：手动运行

如果批处理脚本遇到问题，您可以手动执行以下步骤：

1. **安装Python**：
   - 从官方网站下载Python 3.8或更高版本：https://www.python.org/downloads/windows/
   - 安装时勾选"Add Python to PATH"

2. **安装依赖**：
   - 打开命令提示符
   - 进入项目目录：`cd c:\Users\[您的用户名]\Documents\trae_projects\AI`
   - 运行：`pip install -r requirements.txt`

3. **启动语音助手**：
   - 运行：`python src\main.py`

## 四、常见问题解决

### 问题1：语音识别失败
- 确保麦克风已正确连接并选择
- 检查是否已授予麦克风权限
- 确保网络连接正常

### 问题2：语音合成没有声音
- 检查扬声器是否已正确连接并开启
- 尝试调整音量设置

### 问题3：依赖安装失败
- 检查网络连接
- 确保pip已更新：`pip install --upgrade pip`
- 尝试手动安装失败的包

### 问题4：应用崩溃
- 查看控制台输出的错误信息
- 检查是否有缺失的依赖
- 尝试重新安装依赖
